iOS图片加载过程以及优化

UIImageView加载图片的过程

  1. 从磁盘拷贝数据到内核缓冲区域；

  2. 从内核缓存区域复制数据到用户内存；

  3. UIImage赋值给UIImageView的image时，图像数据会被解码，变成位图数，内存大小为 width height 4bytes （4bytes:每个像素点的大小) 。

  4. CTTransaction捕捉到UIImageView layer树的变化；

  5. 主线程 runloop提交CTTransaction，开始图像渲染。（如果数据没有字节对齐，Core Animation会再拷贝一份数据，进行字节对齐，GPU处理位图数据，进行渲染）。

UIImage几种加载方式的区别

  • imageNamed：在图片第一次渲染到屏幕的时候触发解码，缓存解码之后的数据，缓存在全局内存中，不会随着UIImage的释放而释放。

  • imageWithContentsOfFile: 或 imageWithData: 与imageNamed不同的是会随着UIImage的释放而释放。

几种优化思路

降低采样率

即减少加载UIImage内存的大小,根据imageview实际size来加载，可以减少内存占用。解码后生成CGImage缩略图，再转化为UIImage,然后传给UIImgaeView渲染展示。

异步解码

解码的操作在主线程，比较耗费CPU的资源。可以把耗时的解码操作放入子线程，解码完成之后再回调到主线程刷新，例如SDWebImage。还有更加极限的优化是在子线程解码之后，将解码之后的图片存在磁盘之中，例如FastImageCache。

mmap内存映射

上面提到过从磁盘读取数据时，需要把数据读入内核缓冲区，然后复制到用户内存空间，是比较耗时的。mmap是把文件映射到用户空间的虚拟内存，等同于把文件放入内存，操作中并不消耗物理内存，效率比较高。

字节对齐

UIImageView在渲染之前对于未字节对齐的图像数据拷贝一份，然后进行字节对齐的操作，这一过程消耗也比较多。可以统一管理和优化字节对齐的时机。FastImageCache也针对字节对齐做了优化管理。具体将在下文重点分析FastImageCache。

swift用Stack实现一个队列

基本思路

栈是先进后出，而队列是先进先出，完全不同的顺序，如何做到用栈来实现队列呢？初步的思路是，将栈内容完全反转，这里我们需要两个栈，栈A为原始数据，然后反序压入栈B，再从栈B压出去，那么对于栈A来说数据就是先进先出了

代码实现

首先实现一个stack，定义压栈push、出栈pop、获取内容个数的size等接口。

入队操作，在A栈中压进item。出队操作，在B栈为空的情况下，把A栈压出的item压入B栈，那么位于A栈顶的item将位于B栈底，从而实现AB的顺序反转，如果B栈不为空，则按顺序压出item。从而实现A栈中原始数据的先进先出。

常用的几种排序算法

常用的排序算法

排序算法是最基本的算法，这里主要记录下三种算法：选择排序、冒泡排序和快速排序

选择排序

思路：初始时在序列中找到最大（小）元素，(通常先默认第一个），然后对比其他所有元素，并放到第一位（末尾）。然后对比剩余的元素寻找最大（小）元素，再插入相应位置。

冒泡排序

思路：比较相邻的两个元素，如果left > right,交换两个元素；从头到位遍历一次，此时最大值在最后；以此类推，不断的比较相邻的数值，把大的数字放到后排。

快速排序

思路：通过一趟排序将要排序的数据分割成独立的两部分，其中一部分的所有数据都比另外一部分的所有数据都要小，然后再按此方法对这两部分数据分别进行快速排序，整个排序过程可以递归进行，以此达到整个数据变成有序。

上节实现了链表的一些基本操作，这篇继续查找，合并相关的算法。

链表的实现

单链表正序寻找第N个节点

单链表倒序寻找倒数第N个节点

这个算法的思路是让一个指针先跑，第二个指针后跑，相差k,那么第一个指针跑到结尾的时候，第二个指针也恰好跑到n-k的位置，即倒数k的节点

单链表倒序寻找中间节点

思路和上一个差不多，第一个指针每次跑两步，第二个每次跑一步，那么第一个跑到结束的时候，第二个正好是整个链表长度的一半，即中间节点

寻找一个含环链表的入口节点

思路是第一个指针fast每次跑两步，第二个slow每次跑一步，那么当slow == fast的时候，fast比slow多走n圈（2x=rn+x,x=rn，所以多走的就是环圈的倍数），假设链表开头到环接口的距离是y，此时让fast回到头节点，那么x-y表示slow指针走过的除链表开头y在环中走过的距离，那么slow再走y步，此时fast结点与slow结点相遇，fast == slow ，x-y+y=x = n，即此时slow指向环的入口。

二叉树的原理和用途

为什么要用二叉树

二叉树是一种基本的排序树，用于快速查找。数组搜索比较方便，而插入相对麻烦，链表插入和删除简单，而搜索比较慢，而二叉树是兼顾两者的优点。

二叉树的分类

二叉树因为结构的不同主要分为斜树、满二叉树和完全二叉树，当然还有普通的二叉树，比较基本这里不单独做赘述。

斜树

只有左子树或者只有右子树

满二叉树

所有分支节点都有左子树和右子树，所有叶子结点都在同一层。其有如下特点：1.叶子只能出现在最下一层；2.节点个数为 n = 2^k-1

完全二叉树

完全二叉树和满二叉树的区别是，最后一行可能不完整，但是绝对是右边的连续部分缺失，节点范围是2^(k-1)-1 < n < 2^k-1

求二叉树的深度

//思路：分别计算左子树、右子树的深度，最后取最大值+1即是二叉树深度
func getTresDepth(node: Node?) ->Int{
    var leftDepth = 0
    var rightDepth = 0
    var max = 0
    if (node != nil) {
        leftDepth = getTresDepth(node: node?.leftNode);
        rightDepth = getTresDepth(node: node?.rightNode);
        max = leftDepth > rightDepth ? leftDepth : rightDepth;
        return (max + 1);
    } else {
        return 0;
    }
}
